## REUSABLE ANIMATION COMPONENTS (build these first)

**FadeIn wrapper**
- Framer Motion component using `whileInView`, `viewport={{ once: true, margin: "-80px", amount: 0.2 }}`
- Props: `delay` (default 0), `duration` (default 0.6), `y` (default 16, distance to rise from)
- Easing: `[0.25, 0.1, 0.25, 1]`
- Used to wrap every major section/element on scroll — subtle rise + fade, never slide from the side, never scale-bounce.

**AnimatedCounter**
- Counts a number up from 0 to a target value (e.g. resume score `78`) when it scrolls into view.
- Duration ~1.2s, easeOut, triggers once via `whileInView`.

**AnimatedProgressRing**
- SVG circular progress ring (used for the resume score gauge). The stroke draws in from 0 to the target percentage using `strokeDashoffset` animated by Framer Motion, synced to start at the same time as AnimatedCounter. Green stroke, light gray track underneath. Duration ~1.2s, easeOut.

**MarqueeRow**
- Horizontal infinite-scroll row (used for trust logos). Duplicate the logo list 2–3x for seamless looping. Very slow, constant linear speed (no scroll-linked motion needed here — this is a simple CSS/Framer `animate` loop, not scroll-driven like the 3D portfolio reference). Pauses on hover.

## MOTION PRINCIPLES (apply throughout)

- Every animation triggers once via `whileInView` — never re-plays on scroll up/down repeatedly.
- Nothing bounces, spins unnecessarily, or overshoots. Easing is always smooth ease-out.
- Animations should feel like content settling into place, not "performing" for the user.
- No animation should delay the user from being able to click "Choose File" or "Get Started" — interactive elements must be functional immediately even mid-animation.
- Respect `prefers-reduced-motion` — disable non-essential motion (marquee, floating icon, stagger) for users with that setting, falling back to instant appearance.
